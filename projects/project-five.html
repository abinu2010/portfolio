MiltyKitty

First complete Unity game. A 2D learning project focused on movement, collision, UI, audio, and connecting small mechanics into one playable loop.

Category: Game Projects | Role: Solo Developer | View Repo

About the Game

MiltyKitty is an early Unity 2D game project. Its value in the portfolio is that it shows the starting point of my Unity workflow: scenes, sprites, prefabs, colliders, scripts, UI, audio, and simple gameplay feedback working together in a complete build.

The project focuses on building a simple readable loop rather than a large feature set. It helped me understand how Unity object components, input, collision, scene flow, and UI feedback connect inside an actual playable game.

My Contribution

I built the project as a solo learning piece. I implemented the player controller, collision setup, enemy or hazard interactions, damage feedback, scene flow, menus, score or progress display, health feedback, basic audio hooks, and animation connections.

The main development value was learning how to stop treating systems as isolated scripts. I used MiltyKitty to practise connecting gameplay state to the player, the UI, the scene, and feedback systems so the player understands what is happening.

Technical Work

  • 2D player movement and collision setup.
  • Prefab-based object setup for repeatable scene content.
  • Health and damage feedback linked to UI state.
  • Menu flow and scene transition structure.
  • Audio trigger setup for player actions and feedback.
  • Animation parameter connections for readable state changes.
  • General Unity workflow practice with components, scripts, serialized values, and inspector tuning.
