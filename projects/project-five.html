About the Game

MiltyKitty is a 2D game developed in Unity for my Games and Interactive Systems module. This was my first full Unity project and it was used to build confidence with the editor, prefabs, scenes, input, and the basic workflow of shipping a playable build.

The focus of the game is moment-to-moment responsiveness and readable feedback. The player moves through the level while dealing with enemies, taking damage, and managing health. The game includes UI feedback for score and health, plus audio to reinforce actions and outcomes.

This project was mainly about learning how to connect gameplay systems together: movement, enemy behaviour, damage, UI, and animation, into one loop that actually feels like a game instead of separate mechanics.

My Contribution

I built player movement, enemy AI, and the damage system, and used physics to make interactions feel consistent. I implemented collision handling and tuning so the game plays smoothly rather than feeling random.

I created menus, score tracking, and health systems, and connected UI updates to gameplay events. I also integrated audio and animations to make the game feel alive and easier to understand during play.
