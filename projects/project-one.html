LumberJill

Build, Craft, Care. A cozy woodworking RPG where every tree and trade matters.

Category: Game Projects | Role: Lead Programmer | GitHub

About the Game

LumberJill: The RPG is a 2.5D isometric, session-based mobile management and crafting prototype built in Unity. The player runs a woodworking business by gathering materials, processing resources through machines, crafting products, completing orders, and reinvesting rewards into workshop progression.

The project includes a live lumber economy, where external market data is converted into readable in-game price changes. This gives the management loop an extra layer of timing and decision-making, because the player must think about when to buy, stockpile, and produce.

Crafting uses a blueprint-style grid where processed wood shapes fit into product outlines. Correct placements snap into place, while mistakes waste material and reduce the quality of the final order.

My Contribution

I was the sole programmer for the final-year group project. I built the main gameplay architecture, including the computer/shop interface, the ScriptableObject item, machine, and recipe data pipeline, machine purchasing, and machine placement in the world.

I implemented machine workflows, interactable stations, input/drop zones, configuration panels, saving/loading, and UI state management. I also integrated teammate assets and fixed late-stage prefab, event, and build issues.
