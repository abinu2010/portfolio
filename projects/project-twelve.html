Echoes of The Twin Tombs

Cooperate or stay trapped. A multiplayer VR escape room built in Unity with XR Interaction Toolkit and networked player synchronisation.

Category: Game Projects | Role: VR Gameplay and Interface Systems Programmer | View Repo

About the Game

Echoes of The Twin Tombs is a cooperative VR escape room built in Unity and tested on Meta Quest 3. The experience uses separated player roles, Engineer and Navigator, so one player receives information while the other performs physical actions. The design depends on voice communication, shared puzzle state, controller interaction, and world-space guidance.

The project was not just a set of interactable props. The main challenge was creating a complete multiplayer VR flow where players can connect, choose roles, spawn into different spaces, learn cooperation, solve communication puzzles, and return to the lobby cleanly.

My Contribution

My specialist focus was VR gameplay and interface systems programming. I worked on multiplayer room flow, role selection, role-based spawning, separate lobby spaces, cooperative puzzle logic, world-space UI, warning feedback, reset behaviour, and return-to-lobby flow.

I treated UI as part of the gameplay. In VR, menus and prompts have to sit inside the world where players can read and interact with them naturally. I built the guidance layer around lobby panels, puzzle prompts, telescope mode, diagnostic screens, status text, warnings, and pause controls.

Systems I Built

  • Networked room flow for connecting both players into the same shared experience.
  • Role selection sync so players are assigned different responsibilities instead of both seeing the same gameplay information.
  • Separate lobby spawning and role-based spawn logic using a controlled teleport flow for the XR Origin.
  • Lever and trapdoor puzzle used as the first onboarding test for simultaneous player action.
  • Astral Relay puzzle where the Navigator reads a telescope constellation clue and the Engineer places star orbs into correct sockets.
  • Prism Reactor puzzle using a state-based beam routing system instead of risky real raycast laser reflection.
  • World-space UI and warning feedback to explain tasks inside the headset.
  • Reset and return-to-lobby behaviour so the experience could be replayed and tested reliably.

Design Decisions

The biggest technical choice was to control the full game flow with a reliable structure instead of allowing multiple scripts to move the player or manage role state at the same time. For the Prism Reactor, I chose a state-based beam system because it reduced technical risk while still giving the visual impression of energy routing. For the telescope, I changed the clue from a floating UI panel into a local camera-attached telescope mode so the interaction felt more like using an actual tool.
