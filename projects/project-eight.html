WeatherSys

Real-time procedural weather plugin. An Unreal Engine 5.4 C++ system driven by live OpenWeather API data.

Category: Technical Projects | Role: C++ Plugin Developer | Video

About the Plugin

WeatherSys is a real-time procedural weather plugin developed for Unreal Engine 5.4. It uses live data from the OpenWeather API to drive environmental visuals including lighting, rain, snow, cloud coverage, fog, sunrise, sunset, and night transitions.

The plugin also includes a custom Slate editor panel for assigning scene components, forcing weather states, and testing visual conditions during Play In Editor or Standalone Mode. This makes it useful both as a runtime weather system and as a developer testing tool.

Core Features

  • Real-time weather synchronization through the OpenWeather API.
  • Dynamic sunrise, sunset, day, and night transitions based on real timestamps.
  • Niagara-based rain and snow systems with runtime toggles.
  • Fog density interpolation based on live or forced humidity values.
  • Cloud material color and coverage controlled by cloudiness percentage.
  • Slate-based WeatherSys panel for assigning scene references.
  • Developer override toggles for rain, snow, fog, sunrise, sunset, and night testing.
  • Configurable automatic weather refresh timing, with a default interval of 10 minutes.

Runtime Setup

The WeatherSys panel is opened through Window > WeatherSys after Play In Editor or Standalone Mode starts. Scene components are assigned through the panel while the game is running to avoid null references and runtime crashes.

  • Assignable components include the Volumetric Cloud Actor, Niagara rain and snow systems, Directional Light, Sky Light, and Exponential Height Fog.
  • Lighting controls support configurable day, sunrise, and sunset color presets.
  • Fog controls include minimum density, maximum density, force fog, and humidity override values.
  • The Apply Settings button reinitializes assigned runtime components during testing.

Technical Implementation

The system is entirely C++ driven and does not require Blueprints. The WeatherController actor retrieves weather data at regular intervals and applies the resulting values to lighting, fog, cloud materials, and Niagara precipitation effects.

Cloud visuals use a Dynamic Material Instance on the Volumetric Cloud Component. Rain and snow use GPU-accelerated Niagara systems. Runtime logs confirm applied weather values, forced states, and visual transitions while testing.

Planned Enhancements

  • Environmental sound effects for rain, wind, and thunder.
  • Rain splash particles and wet surface blending.
  • Additional weather types such as storm, haze, and wind zones.
  • Localized regional weather support.
  • Runtime sliders for precipitation intensity.
  • Blueprint API support for in-game scripting and weather events.

Troubleshooting Notes

  • If Apply causes a crash, all Niagara systems and scene actors need to be assigned during runtime.
  • If fog or rain is not visible, humidity, cloudiness, visibility, and Niagara activation settings need checking.
  • If lighting does not change, the assigned Directional Light reference is usually missing or incorrect.
  • If clouds do not update, the Volumetric Cloud Component and Dynamic Material Instance need checking.
