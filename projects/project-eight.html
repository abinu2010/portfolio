WeatherSys

Real-time procedural weather plugin. An Unreal Engine 5.4 C++ system driven by live OpenWeather API data and in-editor overrides.

Category: Technical Projects | Role: C++ Plugin Developer | Video

About the Plugin

WeatherSys is a real-time procedural weather plugin developed for Unreal Engine 5.4. It integrates live weather data from the OpenWeather API to drive environmental visuals such as lighting, precipitation, cloud coverage, fog, sunrise, sunset, and night states.

The system also provides an in-editor Slate panel that lets developers assign scene components, force conditions, simulate lighting states, and reapply settings during Play In Editor or Standalone testing. This makes it useful both as a runtime weather controller and as a developer-facing testing tool.

My Contribution

I built the plugin as a C++ driven technical system. The main work was designing the WeatherController runtime behaviour, connecting API values to scene visuals, exposing editable properties, and building the WeatherSys Slate panel for manual testing.

I also added override controls so weather could be forced during development instead of waiting for a real API condition. This made it easier to test rain, snow, fog, sunrise, sunset, night lighting, humidity response, and cloud coverage inside a controlled scene.

Core Features

  • Real-time weather synchronization through the OpenWeather API.
  • Dynamic sunrise, sunset, day, and night transitions based on real timestamps.
  • Niagara-based rain and snow systems with runtime toggles.
  • Fog density interpolation based on live or forced humidity values.
  • Cloud material colour and coverage linked to cloudiness percentage.
  • Slate-based editor panel for assigning scene references.
  • Developer override toggles for rain, snow, fog, sunrise, sunset, and night states.
  • Configurable automatic weather update interval, with a default of 10 minutes.

Runtime Setup

The WeatherSys panel is opened through Window > WeatherSys after Play In Editor or Standalone Mode starts. Scene components such as the Rain System, Snow System, Fog Actor, Directional Light, Sky Light, and Volumetric Cloud Actor are assigned through the panel while the game is running to avoid null references and runtime crashes.

Technical Implementation

The plugin is entirely C++ driven and does not require Blueprints. Weather data is retrieved at regular intervals, parsed, converted into readable scene values, and then applied to lighting, fog, cloud material instances, and Niagara precipitation systems. Runtime logs confirm applied weather values and forced override states during testing.
