Reactive Containment Chamber

Real-time Unreal shader showcase. A contained sci-fi lab scene focused on reactive liquid, translucent glass, emissive specimen effects, cable animation, floor circuitry, smoke, and a CCTV-style post-process view.

Category: Technical Projects | Role: Technical Artist and Shader Programmer | Video

About the Project

Reactive Containment Chamber is a real-time shader programming project built in Unreal Engine 5.5.4. The scene is a compact sci-fi laboratory centred around a containment tank that holds animated reactive liquid and a pulsing specimen. The goal of the piece was to show multiple shader techniques working together inside one readable environment rather than as isolated material tests.

The scene combines translucent liquid behaviour, glass refraction, emissive procedural effects, animated World Position Offset motion, procedural floor circuitry, Niagara smoke, and a CCTV-style post-process effect. Together, these systems create a contained environment that feels active, monitored, and unstable while still staying readable as a technical showcase.

My Contribution

My main contribution was the technical art and shader implementation. I created the reactive liquid shader using procedural HLSL noise, opacity control, emissive colour blending, and Depth Fade. I built the glass tank material using translucency, Fresnel rim lighting, and refraction so the shell remained readable while still revealing the liquid and specimen inside.

I also developed the pulsing specimen shader so it felt like the energy source of the chamber. That energy was extended through the environment using animated cable wobble with World Position Offset and procedural floor circuit pulses. To strengthen the presentation, I created a CCTV-style post-process shader with screen-space distortion, scanlines, vignette treatment, and colour grading, then connected it to a Blueprint trigger that swapped the player view to a fixed surveillance camera.

I used Material Parameter Collections to control shared values such as tank intensity, liquid speed, glass distortion, cable wobble, circuit speed, and post-process strength. This allowed the materials to react together and made the scene much easier to tune during development.

Technical Breakdown

  • Reactive liquid shader built with procedural HLSL noise, opacity tuning, emissive blending, and Depth Fade.
  • Glass tank material using translucency, Fresnel edge lighting, and refraction.
  • Pulsing specimen shader with emissive vein patterns to drive the chamber focal point.
  • Animated cable movement using World Position Offset.
  • Procedural floor circuit pulses to carry energy through the environment.
  • Niagara smoke to reinforce the containment atmosphere.
  • CCTV-style post-process material with scanlines, distortion, vignette, and colour treatment.
  • Blueprint camera trigger to switch between the normal scene view and the surveillance angle.
  • Material Parameter Collections used to synchronise values across multiple materials.

Performance and Optimisation

I evaluated the scene using Shader Complexity, Material Stats, FPS testing, and GPU profiling. The hardest optimisation issue came from combining translucent materials, post-processing, emissive surfaces, and Niagara smoke in one compact scene. I created quality variants and made visual trade-offs to keep performance stable while preserving the intended sci-fi atmosphere.
