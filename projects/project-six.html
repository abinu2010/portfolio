AI Stealth Heist Prototype

Multi-sense stealth AI. A Unity 3D prototype using FSM guards, NavMesh movement, vision, hearing, smell, bait, and shared alert communication.

Category: Technical Projects | Role: AI and Gameplay Programmer | View Repo

About the Prototype

AI Stealth Heist Prototype is a Unity 3D stealth prototype focused on guard behaviour rather than visual polish. The player controls a first-person character trying to steal an object protected by four guards. The guards patrol, detect, investigate, search, chase, and attack based on multiple perception inputs.

The project began as a 2D stealth idea, but I changed direction after realising that reliable 2D pathfinding would be too risky for the scope. Moving to a 3D Unity scene let me use NavMesh and focus on the AI techniques clearly.

My Contribution

I implemented the AI architecture, player movement, first-person camera control, guard NavMesh routes, field-of-view checks, sound events, smell trails, bait distraction, suspicion levels, global alerts, and FSM state transitions.

The goal was to avoid a basic patrol-chase-attack enemy. Each guard reacts through layered perception and can escalate behaviour through suspicion, investigation, searching, chasing, and attacking. A global alert system allows one guard detection to affect the wider group.

Technical Work

  • Player movement with walk, sprint, and crouch speed differences.
  • First-person mouse look with vertical angle clamping.
  • Guard Field of View using range checks, angle checks, raycasts, and obstacle layer masks.
  • Sound Bus singleton where noise events send position, radius, and intensity to subscribed guards.
  • Different hearing reactions for faint, medium, and strong sounds.
  • PlayerNoiseEmitter behaviour where sprinting is louder and crouching is quieter.
  • Bait system used to attract guards without immediately triggering the same response as direct player detection.
  • Smell trail logic to let guards follow traces instead of only reacting to sight.
  • Global alert communication so guards can share suspicion and react as a group.

Evaluation

The prototype demonstrates how simple AI techniques become stronger when combined. NavMesh movement, FOV, hearing, smell, bait, and alert sharing make the guards feel more reactive than a single-state patrol enemy. The main limitation is scope: the level is simple and the guard type is duplicated, but the AI behaviour is clear enough to showcase FSM design and debugging.
