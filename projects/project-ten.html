Boat In A Bottle

48-hour Small Boats jam. A 2D ship-in-a-bottle prototype about placing boat parts without breaking the glass.

Category: Game Jams and Experiments | Role: Main Developer | GitHub

About the Jam

Boat In A Bottle was built for the Personal Development Week Jam, a 48-hour game jam based on the theme Small Boats. The project became a 2D ship-in-a-bottle prototype where the player places boat parts inside a floating glass bottle without touching the bottle edges.

The fail state is simple: if the piece touches the bottle, the bottle breaks and the game restarts. The success state is completing the boat inside the bottle, after which the boat transforms into a larger version. The original plan had three levels, but the scope was cut down to one complete challenge to keep the prototype achievable.

My Contribution

  • Worked as the main developer while two teammates focused on art assets.
  • Built the core playable loop from start to finish.
  • Created the bottle wave motion and drift system so the bottle felt like it was floating on water.
  • Connected the win and lose conditions to the boat-piece placement mechanic.
  • Built the main menu, play flow, quit flow, scene transitions, and simple audio manager.

Technical Notes

  • Replaced the first CapsuleCollider2D setup with a PolygonCollider2D so the bottle opening worked more accurately.
  • Adjusted the water visual setup after sprite assignment problems caused only the wave crest to appear.
  • Changed the wave presentation to use a line-rendered crest for clearer water movement.
  • The biggest lesson was to finish the full loop first instead of over-polishing a single visual mechanic too early.
