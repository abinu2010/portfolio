SpiderSquish

Dash to defeat. A top-down 2D Unity survival project built with OOP, enemy spawners, projectiles, score gates, and power-ups.

Category: Game Projects | Role: Gameplay Programmer | View Repo

About the Game

SpiderSquish is a top-down 2D Unity project built for Object-Oriented Programming coursework. The player moves through a forest arena, survives enemy projectiles, collects health boosts, defeats spiders with dash attacks, reaches the required score, reveals the key, and exits through the door.

The core loop is simple and readable: move, dodge, dash through enemies, manage health, collect power-ups, increase score, unlock the key, and escape. Different enemy setups create pressure through speed, shooting range, and fire rate.

My Contribution

I implemented the core gameplay systems: player movement, dash attack, enemy behaviour, projectile shooting, enemy health, collision damage, power-up spawning, score tracking, key reveal, door unlock, UI, and audio triggers.

The project was structured around reusable object-oriented systems. Enemies, power-ups, audio categories, spawners, inventory, score, key, and door behaviour were separated into focused scripts so each part of the game could be tested and changed without breaking the rest.

Technical Work

  • Enemy movement and rotation using Rigidbody2D physics and vector calculations toward the player.
  • Projectile shooting with distance checks, angle calculations, and damage behaviour.
  • Dash collision logic where enemies are only defeated after the required number of dash hits.
  • Enemy spawners using coroutines, spawn intervals, prefab arrays, and random selection.
  • Power-up spawners using coroutine timing and random prefab selection for different health boosts.
  • Audio manager using AudioClip arrays, categories, random clip selection, and null safety checks.
  • ScoreManager singleton that tracks score, updates UI, and reveals the key after the required threshold.
  • PlayerInventory and KeyDoor logic to control the final exit condition.
